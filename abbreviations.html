Abbreviations¶

Usage of abbreviations in scripts is covered here: Body Abbreviations. This document covers the module itself.

Module Documentation¶

class lacuna.abbreviations.Abbreviations(client, vardir=None)¶

Bases: object

Saves personal abbreviations for often-long body names.

Parameters:
  • client (lacuna.clients.Member) – The client we’re currently connected as.
  • vardir (str) – The path to the Monty var/ directory. Defaults to CALLING_SCRIPT/../var/.
delete(name)¶

Deletes an abbreviation assignment, given the assignment’s full name.

Parameters:name (str) – Full name of a body; its abbreviation will be removed.
Raises:KeyError – If no abbreviation exists for name.
get_abbrv(name)¶

Gets the abbreviation for a full name.

Parameters:name (str) – Full name of a body
Returns:abbrv – The abbreviation set for that body
Return type:str
Raises:KeyError – if no abbreviation is set for name.
get_name(abbrv)¶

Gets the full name for an abbreviation

Parameters:abbrv_or_name (str) – Either a pre-set abbreviation or a body full name. If the given string is not found in the abbreviations database, it’s assumed to be a full body name and returned as-is.
Returns:full_name – If the argument was a set abbreviation, returns
the full unabbreviated name. If the argument was not a set abbreviation, but matches a body name (either colony or space station), that argument is returned as passed in.
Return type:str
Raises:KeyError – If the abbrv passed in is neither a set abbreviation nor a valid body name.
list_abbrvs()¶

Get a list of currently-set abbreviations

Returns:list – Alpha-sorted list of abbreviations
Return type:list of str
list_names()¶

Get a list of full body names for which there are abbreviations set

Returns:list – Alpha-sorted list of names
Return type:list of str
save(name, abbrv)¶

Saves an abbreviation assignment.

Parameters:
  • name (str) – The full name
  • abbrv (str) – The abbreviation
show(name)¶

Produces a report on a single stored abbreviation.

The report is displayed on STDOUT. If we’re reporting on ‘all’ abbreviations, the report is displayed via a pager.

show_all()¶

Produces a report on all currently-stored abbreviations.

The report is displayed on STDOUT via a pager.
